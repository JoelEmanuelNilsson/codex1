# Blueprint Support Files

Use this folder only when the mission is too large or too coupled for a single
compact `PROGRAM-BLUEPRINT.md`.

`PROGRAM-BLUEPRINT.md` remains the canonical index. Any split-out detail here
should be referenced from that index and should not silently replace it.

## Suggested Expansion Files

- `truth-register.md`: verified facts, assumptions, contradictions, and unknowns
- `system-map.md`: touched surfaces, boundaries, and hidden couplings
- `decision-obligations.md`: obligation records, evidence, and selection state
- `proof-matrix.md`: proof rows, evidence classes, and ownership
- `execution-graph.md`: runnable nodes, waves, and safe-wave rules

## Notes

- Split files only when the added structure makes the mission easier to reason
  about.
- Keep all files mission-specific and revision-aware.
- When a later replan supersedes detail here, update the canonical index first
  and then the split file.
